J.A.R.V.I.S. AI - Privacy & Data Policy

J.A.R.V.I.S. is a fully self-hosted, local-first AI Desktop Assistant. This policy explains what data is accessed, how it is processed, and where it is stored. Because J.A.R.V.I.S. runs entirely on your local machine, your data remains under your absolute control. No personal data is collected or stored on external servers by developer endpoints.

1. Local Storage & Databases

All configuration, state, and user logs are kept locally on your computer. The application utilizes the following SQLite databases and local configuration files:

  • jarvis_user_data.db: Stores user settings, personalizations, local device states, and credential mappings.
  • jarvis_bot_data.db & jarvis_bot_logs.db: Handles message history, logs, and queue states for the integrated Telegram bot.
  • reminders.db: Stores custom user alarms, tasks, and notification schedules.
  • token.json & token.pickle: Stores encrypted local Google OAuth authorization tokens.
  • .env: Stores sensitive developer keys (e.g., Cohere/Groq API keys, Sinric Pro credentials, and local networking hosts).

2. Hardware Access & Sensor Monitoring

To perform desktop automation and status reports, J.A.R.V.I.S. requests access to specific local hardware components:

  • Microphone Input: Audio streams are processed in-memory using Python's speech_recognition and pyaudio libraries. Audio fragments are converted to text and immediately discarded; voice files are never saved to disk.
  • Camera & Vision: Camera frames are scanned in-memory by a local YOLOv8 neural network. No video frames, screenshots, or camera streams are saved on disk or sent over the network.
  • macOS System Sensors: Compiled locally from SensorsMonitor.swift to query hardware status (temperature, battery, fan speed) via macOS CoreFoundation/IOKit.
  • Clipboard Monitoring: Monitors copied text to enable smart automation triggers.

3. Third-Party API Integrations

When J.A.R.V.I.S. needs external intelligence or cloud integrations, it interacts directly with third-party servers using your personal credentials:

  • Google Workspace APIs (Gmail, Tasks, Drive, Calendar, YouTube): All data is processed locally inside Python event loops and is never sent to third parties.
  • Large Language Models (Groq, Cohere): Text queries are sent to Groq/Cohere APIs using the keys in your .env.
  • Sinric Pro: Used to toggle smart home switches.

4. Revoking Scopes & Data Deletion

Since all data is stored locally on your device, you have complete control to delete it by removing token.json, your database files, or your API keys from the .env file.
